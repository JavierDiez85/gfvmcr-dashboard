3. Seguridad — Analisis Critico

Vulnerabilidades en tu ERP Grupo

ProblemaSeveridadDetalle
SHA-256 sin salt CRITICA Misma contrasena = mismo hash siempre. Vulnerable a rainbow tables. Archivo: login.js:7-11
Sin escapeHtml() CRITICA XSS posible via nombres de usuario. Un nombre como <img onerror="..."> ejecutaria codigo malicioso
localStorage expuesto CRITICA Cualquier usuario puede abrir DevTools y ver TODOS los hashes de passwords de todos los usuarios
Bypass de permisos via consola ALTA Un usuario puede escribir sessionStorage.setItem('vmcr_session', '{"rol":"admin"}') y obtener acceso admin
Sin auditoria ALTA No hay registro de quien creo, elimino o modifico usuarios

function escapeHtml(s) {
  return String(s || "")
    .replaceAll("&", "&amp;")
    .replaceAll("<", "&lt;")
    .replaceAll(">", "&gt;")
    .replaceAll('"', "&quot;")
    .replaceAll("'", "&#039;");
}

Calculo de comision (Emiliano — renders.js)

function feeFromRow(r) {
  const net = Number(r.monto || 0);
  const rate = COMMISSION_RATES[getTypeKeyFromRow(r)];
  const commission = net * rate;              // Comision base
  const iva = commission * 0.16;              // 16% IVA
  const totalFee = commission + iva;          // Fee total
  const liquidated = net - totalFee;          // Neto a pagar al cliente
  return { net, rate, commission, iva, totalFee, liquidated };
}

Clasificacion automatica de tipo de tarjeta

function getTypeKeyFromRow(r) {
  const marca = String(r.marca_tarjeta || "").toLowerCase();
  const metodo = String(r.metodo_pago || "").toLowerCase();
  if (marca.includes("amex")) return "amex";
  if (metodo.includes("debito")) return "debito";
  if (metodo.includes("credito")) return "credito";
  if (metodo.includes("internacional")) return "internacional";
  return "credito";
}

13. Plan de Accion y Prioridades

URGENTE — Seguridad

  1. Migrar autenticacion a Supabase Auth — Elimina SHA-256 sin salt, implementa bcrypt server-side con manejo automatico de sesiones JWT
  2. Agregar escapeHtml() — Implementar funcion de escape HTML y aplicarla en TODA salida de datos de usuario para prevenir XSS
  3. Filtrar datos por usuario — Agregar .eq("user_id", userId) en todos los queries a Supabase para scoping de datos
  4. Expiracion de sesion — Implementar timeout de inactividad (30 min sugerido)

ALTO IMPACTO — Funcionalidad

  1. 6 roles con jerarquia — director > admin > supervisor > comercial > comisionista > viewer
  2. Permisos por usuario individual — El director puede personalizar acceso por pagina para cada usuario
  3. Motor de validacion de pagos — Deteccion de duplicados, pagos vencidos, excesos, con tolerancia configurable
  4. Calendario de dias habiles — Festivos Banxico + cutoff 22:59 + calculo automatico de fecha de pago
  5. Override de comisiones por cliente — Tasas personalizadas por cliente, con scoping por director

MEJORA TECNICA — Mantenibilidad

  1. Partir archivos grandes — terminales.js (2,840 lineas) en 5-6 archivos especializados
  2. Centralizar configuracion — Un solo config.js con todas las constantes del sistema
  3. Mover estilos inline a CSS — Clases reutilizables en vez de style="..." en JavaScript
  4. Event delegation — Un listener centralizado en vez de onclick="..." inline en cada boton
  5. Funciones puras para calculos — Separar logica de negocio del renderizado DOM

NICE TO HAVE

  1. Exportacion CSV/PDF/Clipboard — Exportar comisiones, terminales, pagos en multiples formatos
  2. Void/anulacion con auditoria — Marcar pagos como anulados en vez de borrarlos
  3. Split de pagos fiscal/tarjeta — Configurable por cliente con preferencias guardadas
  4. Charts expandibles con descarga PNG — Click para fullscreen + boton de descarga
  5. Auto-refresh toggle — Actualizacion automatica de datos con boton on/off
